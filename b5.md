Mã vé hợp lệ phải thỏa mãn các quy tắc sau:
1. Không được null, không được rỗng (trim trước khi xử lý).
2. Bắt đầu bằng tiền tố "BUS-" (phân biệt chữ hoa/thường).
3. Sau "BUS-" là đúng 2 ký tự chữ in hoa (mã tỉnh, ví dụ: HN, SG, DN).
4. Tiếp theo là dấu gạch ngang "-" và đúng 6 chữ số (YYMMDD).
5. Ngày tháng phải hợp lệ (ví dụ: không có ngày 32/13, năm nhuận được xử lý đúng).
6. Ngày đi phải lớn hơn hoặc bằng ngày hiện tại (lấy theo hệ thống, múi giờ mặc định của JVM).

Yêu cầu kỹ thuật:
- Sử dụng java.time.LocalDate và DateTimeFormatter cho mọi thao tác ngày tháng (không dùng SimpleDateFormat hay Date).
- Xử lý ngoại lệ (DateTimeParseException) một cách an toàn, trả về false thay vì ném exception.
- Viết comment giải thích các bước kiểm tra.
- Không sử dụng thư viện bên ngoài.

java
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.time.format.DateTimeParseException;

public class TicketValidator {

    private static final String PREFIX = "BUS-";
    private static final DateTimeFormatter DATE_FORMATTER = DateTimeFormatter.ofPattern("yyMMdd");

    public static boolean isValidTicketCode(String ticketCode) {
        if (ticketCode == null || ticketCode.trim().isEmpty()) {
            return false;
        }
        String code = ticketCode.trim();

        // 1. Kiểm tra tiền tố
        if (!code.startsWith(PREFIX)) {
            return false;
        }

        // 2. Cắt bỏ tiền tố và tách phần còn lại
        String rest = code.substring(PREFIX.length());
        if (rest.length() != 10) { // 2 chữ tỉnh + '-' + 6 số
            return false;
        }

        // 3. Tách mã tỉnh (2 chữ) và phần ngày (6 số)
        String provinceCode = rest.substring(0, 2);
        if (!provinceCode.matches("[A-Z]{2}")) {
            return false;
        }
        if (rest.charAt(2) != '-') {
            return false;
        }
        String datePart = rest.substring(3);
        if (!datePart.matches("\\d{6}")) {
            return false;
        }

        // 4. Kiểm tra ngày hợp lệ và không quá khứ
        try {
            LocalDate travelDate = LocalDate.parse(datePart, DATE_FORMATTER);
            LocalDate today = LocalDate.now();
            return !travelDate.isBefore(today);
        } catch (DateTimeParseException e) {
            return false;
        }
    }
}

Phân tích phản biện:

Sau khi xem xét code, tôi phát hiện 3 vấn đề tiềm ẩn:

Nguy cơ NullPointerException: Dù đã kiểm tra ticketCode == null, nhưng ticketCode.trim() sau đó vẫn có thể gây lỗi nếu ticketCode là null vì đã check trước, nên an toàn. Tuy nhiên, nếu ticketCode là chuỗi rỗng, trim() trả về "", isEmpty() đúng → trả false, không sao. Thực ra không có NPE ở đây.

Vấn đề múi giờ và ngày hệ thống: LocalDate.now() lấy theo múi giờ mặc định của JVM. Nếu server đặt múi giờ không phải UTC+7 (múi giờ Việt Nam), thì so sánh ngày đi có thể bị lệch 1 ngày. Ví dụ: lúc 23:00 UTC+7 ngày 15/06, nhưng JVM ở UTC+0 vẫn là 15/06, nhưng thực tế ở Việt Nam đã là 16/06, gây sai. Cần chỉ định rõ ràng ZoneId cho Việt Nam hoặc yêu cầu truyền thời gian hiện tại từ hệ thống.

Xử lý năm hai chữ số: DateTimeFormatter.ofPattern("yyMMdd") sẽ giải thích năm 26 là 2026, nhưng với năm 70 sẽ là 1970 hay 2070? Java mặc định dùng phạm vi 2000-2099 (khi 70 → 1970, 30 → 2030). Điều này có thể gây nhầm lẫn khi vé có ngày đi sau 2070 hoặc trước 1970, mặc dù thực tế có thể không xảy ra. Tuy nhiên, cần xử lý rõ ràng hơn: nên yêu cầu định dạng YYMMDD với giả định năm 2000+ (ví dụ 26 → 2026). Nhưng nếu vé in ngày 01/01/99, sẽ thành 1999, không đúng với thực tế (nếu vé được phát hành từ 2020 trở đi). Nên quy định cụ thể rằng năm trong vé thuộc khoảng 2024-2099 hoặc dùng định dạng rõ ràng hơn.

Đề xuất sửa:

Dùng LocalDate.now(ZoneId.of("Asia/Ho_Chi_Minh")) để lấy ngày theo giờ Việt Nam.

Thêm kiểm tra khoảng năm hợp lý (ví dụ từ 2024 đến 2099) hoặc sử dụng DateTimeFormatter với giả định năm 21xx nếu cần. Ở đây tôi đề xuất chấp nhận năm từ 2024-2099 vì vé có hiệu lực trong tương lai gần.

Cũng cần kiểm tra thêm rằng ngày đi không được quá xa (ví dụ > 10 năm) để tránh lỗi logic.

java
import java.time.LocalDate;
import java.time.ZoneId;
import java.time.format.DateTimeFormatter;
import java.time.format.DateTimeParseException;

/**
 * Lớp kiểm tra mã vé xe khách theo quy tắc GreenBus.
 * Sử dụng java.time hiện đại, xử lý múi giờ Việt Nam.
 */
public class TicketValidator {

    private static final String PREFIX = "BUS-";
    private static final ZoneId VIETNAM_ZONE = ZoneId.of("Asia/Ho_Chi_Minh");
    private static final DateTimeFormatter DATE_FORMATTER = DateTimeFormatter.ofPattern("yyMMdd");

    /**
     * Kiểm tra mã vé hợp lệ.
     * @param ticketCode mã vé cần kiểm tra
     * @return true nếu mã hợp lệ và ngày đi không quá khứ (tính theo giờ Việt Nam)
     */
    public static boolean isValidTicketCode(String ticketCode) {
        // 1. Kiểm tra null và rỗng
        if (ticketCode == null || ticketCode.trim().isEmpty()) {
            return false;
        }
        String code = ticketCode.trim();

        // 2. Kiểm tra tiền tố
        if (!code.startsWith(PREFIX)) {
            return false;
        }

        // 3. Tách phần còn lại sau "BUS-"
        String rest = code.substring(PREFIX.length());
        if (rest.length() != 10) { // 2 tỉnh + '-' + 6 số
            return false;
        }

        // 4. Kiểm tra mã tỉnh (2 chữ in hoa)
        String provinceCode = rest.substring(0, 2);
        if (!provinceCode.matches("[A-Z]{2}")) {
            return false;
        }

        // 5. Kiểm tra dấu gạch ngang và ngày tháng
        if (rest.charAt(2) != '-') {
            return false;
        }
        String datePart = rest.substring(3);
        if (!datePart.matches("\\d{6}")) {
            return false;
        }

        // 6. Phân tích ngày, kiểm tra hợp lệ và không trong quá khứ (theo giờ Việt Nam)
        try {
            LocalDate travelDate = LocalDate.parse(datePart, DATE_FORMATTER);
            // (Tuỳ chọn) Giới hạn năm hợp lý để tránh nhầm lẫn: 2024-2099
            int year = travelDate.getYear();
            if (year < 2024 || year > 2099) {
                return false;
            }

            LocalDate today = LocalDate.now(VIETNAM_ZONE);
            return !travelDate.isBefore(today);
        } catch (DateTimeParseException e) {
            return false;
        }
    }
}
